# Excel Deliverable

Use this skill when the requested output should be a real Excel workbook.

## Best-fit cases

- model output from `3-statement-model`, `dcf-model`, `lbo-model`, or `merger-model`
- workbook packaging after `datapack-builder`
- comparable-company workbook after `comps-analysis`
- tracker or working file after `deal-tracker`
- clean handoff workbook after `customer-analysis-pack`

## Tooling preference

Prefer the bundled MiniMax-derived Excel skill shipped inside this plugin:

- `minimax-xlsx`

If that bundled path is not available for any reason, fall back to the standard `xlsx` workflow already present in the environment.

This plugin now vendors the MiniMax XLSX skill as a convenience layer. If the host already has a compatible Excel / XLSX skill installed, that is also fine, but a separate MiniMax install should no longer be required for Excel output after this plugin is installed.

Do not use `which`, PATH probing, or same-name executable checks as the decision rule. A host may expose the Excel capability as a routed skill without installing a shell binary called `minimax-xlsx`.

The key rule is:

- create or update a real `.xlsx`
- preserve formulas when the workbook is analytical
- keep formatting intentional and minimal

## Workflow

### Step 1: Identify workbook role

Decide whether the workbook is mainly:

- a calculation model
- an analysis pack
- a tracker
- a presentation support workbook

This determines how much formula logic versus presentation formatting is needed.

### Step 2: Pull source data from plugin skills

Start from outputs generated by:

- `3-statement-model`
- `comps-analysis`
- `datapack-builder`
- `dcf-model`
- `lbo-model`
- `merger-model`
- `deal-tracker`

If the user supplied an existing workbook, treat it as the source of truth for structure.

### Step 3: Build or refresh the workbook

Requirements:

- formulas for calculated fields where appropriate
- clear tab naming
- units stated at the top of key sheets
- clean separation of inputs, calculations, and outputs
- no silent hardcoding of derived values in model sections
- do not fail just because a shell-level MiniMax executable is absent

### Step 4: Pair with other office outputs when needed

Common handoffs:

- to `word-deliverable` for memo packaging
- to `ppt-deliverable` for slide-ready outputs
- to `pdf-deliverable` for distribution copies

## Output standard

Deliver:

1. a real `.xlsx`
2. a short note on sheet structure
3. any assumptions or manual inputs that still need user confirmation

## Quality checklist

- workbook opens cleanly
- formulas and linked sections still work
- key tabs are named clearly
- no broken references or obvious calc errors remain
- the file is useful as a working banker artifact, not just a dump
